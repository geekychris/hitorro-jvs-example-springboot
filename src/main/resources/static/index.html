Object Retrieval

Translate → Enrich → Index Pipeline

Pipeline: Load dataset → Translate MLS fields to target languages (Ollama) → Enrich with NLP (segmentation, NER, stemming) across all languages → Store in RocksDB KVStore + Index in Lucene. The index stores only projected fields for search; full documents are retrieved from the KV store.

Object Retrieval & Search

Multi-index retrieval pipeline (hitorro-retrieval) with pluggable search providers, sort-aware result merging, AI summarization, and KVStore document fetch. Single index: full pipeline (search → fetch → enrich → paginate → facets → summarize). Multiple indexes: CompositeSearchProvider fans out to per-index providers, merges results via Score, RRF, or Field sort.
